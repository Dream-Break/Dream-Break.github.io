## 起因

我的博客最早是 2021 年用 Hexo + NexT 主题搭的，当时写了几篇 Java 并发相关的文章，然后就……一直没更新。

时隔四年再看，Hexo 已经不太流行了，NexT 主题虽然经典但显得有些老气。正好最近一直在用 AI 处理各种问题，突发奇想：**能不能让 AI 帮我把博客重建一遍？**

结果还真成了。

## 技术选型

AI 推荐了 **Astro** 作为新框架，理由是：

- 2024-2025 年最火的静态站点生成器
- 构建速度极快（比 Hexo 快得多）
- 默认零 JavaScript，页面加载飞快
- 支持 Markdown/MDX，写文章体验一致
- 组件化开发，自定义主题方便

对比一下老博客的技术栈：

| | 旧博客 | 新博客 |
|---|---|---|
| 框架 | Hexo 5.4 | Astro 6.3 |
| 主题 | NexT Gemini | 自定义 |
| 部署 | GitHub Pages (静态文件) | GitHub Pages (Actions 构建) |
| 评论 | 无 | Giscus (GitHub Discussions) |
| 构建时间 | ~10s | ~1.6s |

## 迁移过程

整个过程大概分这么几步：

### 1. 初始化项目

AI 直接在我的仓库上新建了分支，用 `create-astro` 初始化了一个 blog 模板，然后把默认的英文内容全部替换成中文。

### 2. 迁移文章

原来的 6 篇文章是 Hexo 生成的 HTML，AI 从 git 历史里把原始内容提取出来，重新整理成 Markdown 格式。frontmatter 也从 Hexo 格式转成了 Astro 的 schema。

### 3. 功能开发

这是最有意思的部分。我基本上就是提需求，AI 负责实现：

> "原来有的功能都得有吧"

于是 AI 分析了旧博客的所有功能（侧边栏、TOC、标签云、归档、搜索、Live2D……），然后一个个在新框架里实现。

> "评论加上吧"

加了 Giscus。

> "看板娘太大了，放右下角"

调了。

> "右下角和回到顶部重叠了"

又挪到左下角了 😂

> "加个打字机效果和粒子背景"

加了。

整个过程就像跟一个前端工程师结对编程，只不过这个"工程师"不会累、不会烦，而且速度飞快。

### 4. 部署

配置了 GitHub Actions，push 到 master 自动构建部署。中间还踩了个坑——workflow 里写的触发分支是 `astro-rebuild`，但 GitHub Pages 的环境保护规则只允许 `master` 部署，改一下就好了。

## 最终效果

新博客的功能清单（全部由 AI 实现）：

**基础功能：**
- 暗色/亮色主题切换
- 主题色切换（6种颜色）
- 衬线/无衬线字体切换
- 响应式布局
- RSS 订阅

**文章功能：**
- 文章目录（TOC）自动生成
- 阅读进度条
- 字数统计 & 阅读时长
- 代码块复制按钮 & 语言标签
- 图片灯箱放大
- 上/下篇导航
- Giscus 评论系统
- 文章置顶
- Markdown 脚注

**导航功能：**
- 标签云
- 分类页
- 归档（时间线）
- 友链页
- 全站搜索（Ctrl+K）

**趣味功能：**
- Live2D 看板娘
- 首页打字机 slogan
- 粒子背景动画
- 控制台彩蛋
- Konami Code 撒花彩蛋
- 网站运行时间
- 不蒜子访问统计

## 感受

说实话，整个过程比我想象的顺利得多。以前搞博客主题，光调 CSS 就能调一天，现在基本上说一句话就搞定了。

AI 不是万能的——它偶尔会犯错（比如目录把评论标题也收进去了、锚点跳转偏移等），但好处是你只需要**描述问题**，它就能定位并修复。

对于程序员来说，AI 最大的价值不是替代你写代码，而是把那些**你知道怎么做但懒得做**的事情快速搞定。

比如这个博客，如果让我自己从头写，可能一个周末都搞不完。但有 AI 帮忙，一个下午就全部搞定了，而且功能比我自己写的还多。

## 后记

如果你也想用 AI 重建博客，几个建议：

1. **先想清楚需求再动手** — 不然会改来改去浪费时间
2. **给 AI 参考** — 告诉它你旧博客有什么功能，让它对标实现
3. **分步骤来** — 先搭骨架，再加功能，最后调细节
4. **不要怕提要求** — "看板娘说话字太小" "目录跳转不准" 这种细节直接说就行

工具就是拿来用的，用好了能省很多时间。省下来的时间干嘛？当然是多写几篇文章啦 😄
